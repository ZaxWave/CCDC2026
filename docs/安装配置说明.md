| 组件 | 最低版本 | 推荐版本 |
|------|---------|---------|
| Python | 3.10 | **3.11** |
| Node.js | 18.x | **20.x LTS** |
| PostgreSQL | 14 | **16** |

## 2. Python 环境搭建

### 2.1 创建 Conda 虚拟环境

```bash
conda create -n lightscan python=3.11 -y
conda activate lightscan
```

### 2.2 安装 PyTorch（按显卡型号选择）

```bash
# RTX 40 系列（CUDA 12.4）
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu124

# RTX 50 系列 Blackwell（须用 Nightly，CUDA 12.8）
pip install --pre torch torchvision --index-url https://download.pytorch.org/whl/nightly/cu128

# 无 GPU / CPU 模式
pip install torch torchvision
```

验证安装：

```bash
python -c "import torch; print(torch.__version__, torch.cuda.is_available())"
```

### 2.3 安装项目依赖

```bash
# 在项目根目录执行
pip install -r requirements.txt
```

> **注意**：首次运行视频 OCR 模式时，PaddleOCR 将自动下载约 130 MB 模型文件，请确保网络畅通。

## 3. 数据库初始化

### 3.1 安装并启动 PostgreSQL

**Windows：** 从 https://www.postgresql.org/download/windows/ 下载安装包，安装时记录超级用户（postgres）密码。

**Linux：**
```bash
sudo apt install postgresql postgresql-contrib -y
sudo systemctl start postgresql
```

### 3.2 创建数据库

```bash
# 进入 psql 控制台
psql -U postgres

# 在 psql 内执行
CREATE DATABASE lightscan;
\q
```

### 3.3 配置后端环境变量

在 `src/backend/` 目录下新建 `.env` 文件：

```env
# 数据库连接（替换为实际用户名和密码）
DATABASE_URL=postgresql://postgres:your_password@localhost:5432/lightscan

# JWT 签发密钥（可随机生成 32 位字符串）
SECRET_KEY=your-secret-key-32chars-here

# DeepSeek API 密钥（从 platform.deepseek.com 获取）
DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxx
```

> 数据表（`users` / `disease_clusters` / `disease_records`）将在首次启动后端时**自动创建**，无需手动建表。

### 3.4 初始化管理员账户（可选）

后端启动后，访问 `http://localhost:8000/docs`，展开 `POST /api/v1/auth/register`，注册第一个管理员账号（role 填 `admin`）。

## 4. 后端服务启动

```bash
conda activate lightscan
cd src/backend
uvicorn app.main:app --reload --port 8000
```

启动成功后控制台输出：

```
INFO:     Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
INFO:     Application startup complete.
```

**API 交互式文档：** http://localhost:8000/docs  
**健康检查：** http://localhost:8000/

## 5. Web 前端启动

### 5.1 配置高德地图密钥

在 `src/frontend/` 目录下新建 `.env` 文件：

```env
VITE_AMAP_KEY=你的高德JS_API_Key
VITE_AMAP_SECURITY_CODE=你的安全密钥
```

> 高德密钥申请地址：https://console.amap.com/dev/key/app  
> 服务平台选择「Web端（JS API）」，同时开启安全码。

### 5.2 开发模式运行

```bash
cd src/frontend
npm install
npm run dev
```

访问：http://localhost:5173

### 5.3 生产构建（由 FastAPI 托管）

```bash
npm run build
```

构建产物输出至 `src/frontend/dist/`，FastAPI 启动时自动挂载为静态文件，此后直接访问 http://localhost:8000 即可。

## 6. 移动端运行

### 6.1 安装依赖

```bash
cd src/mobile
npm install
```

### 6.2 H5 开发预览（浏览器）

```bash
npm run dev:h5
```

访问：http://localhost:10086  
在浏览器按 `F12` → 切换手机设备模式（推荐 iPhone 12 Pro，375×844）。

### 6.3 微信小程序构建

```bash
npm run build:weapp
```

1. 打开**微信开发者工具**，选择「导入项目」
2. 目录选择 `src/mobile/dist/weapp/`
3. 填入自己的小程序 AppID（测试号可在微信公众平台申请）
4. 点击「预览」扫码在真机上测试

> **注意**：微信开发者工具调试基础库请选择稳定版（建议 3.4.x），灰度版 3.15.x 存在已知超时兼容问题。

## 8. 常见问题

**Q：`psycopg2` 安装失败（Windows）**  
A：改用二进制包：`pip install psycopg2-binary`

**Q：PaddleOCR 下载模型超时**  
A：在 `.env` 中设置代理，或手动下载模型放置到 `~/.paddleocr/` 目录。

**Q：高德地图显示「安全密钥未配置」**  
A：确认 `VITE_AMAP_SECURITY_CODE` 已填写，且 `window._AMapSecurityConfig` 在地图初始化前赋值。

**Q：GPU 不被识别（`torch.cuda.is_available()` 返回 False）**  
A：确认 CUDA 版本与 PyTorch 安装命令中的 `cu124` / `cu128` 匹配，可通过 `nvidia-smi` 查看驱动支持的最高 CUDA 版本。

**Q：视频推理长时间无响应**  
A：视频较长时后端处理时间可能超过 2 分钟，属正常现象；可在视频配置弹窗中增大采样间隔（帧间距 ↑）以加速。
